🔒 AI 보안 이슈

LiteLLM 공급망 공격 — PyPI 전체 패키지 격리 CRITICAL

월 9,700만 다운로드의 LiteLLM 1.82.7~1.82.8 버전에서 크리덴셜 탈취 멀웨어가 발견되었습니다. TeamPCP 그룹이 Trivy(보안 스캐너) CI/CD 파이프라인을 먼저 침투한 후 LiteLLM 메인테이너의 PyPI 계정을 탈취했습니다. 멀웨어는 SSH 키, AWS/GCP/Azure 크리덴셜, K8s 시크릿을 수집하며, import 없이도 .pth 파일로 자동 실행됩니다. PyPI에서 전체 버전이 격리 조치되었습니다.

💡 백엔드 개발자 포인트 — LiteLLM을 LLM 프록시로 사용 중이라면 즉시 의존성 확인 필수. pip show litellm으로 버전 확인 후, 1.82.7/1.82.8이면 환경 격리·크리덴셜 로테이션을 수행하세요. K8s 환경이라면 privileged pod 배포 여부도 점검해야 합니다.
