Langflow CVE-2026-33017 — CVSS 9.3 치명적 RCE, 공개 20시간 만에 실제 공격 CVSS 9.3

AI 워크플로우 빌더 Langflow에서 치명적 비인증 원격 코드 실행(RCE) 취약점이 발견되었습니다. POST /api/v1/build_public_tmp/{flow_id}/flow 엔드포인트가 사용자 입력의 Python 코드를 샌드박싱 없이 서버에서 실행합니다. 3월 17일 공개 후 20시간 만에 Sysdig가 실제 공격을 탐지했으며, PoC조차 없는 상태에서 공격자들이 권고문만으로 익스플로잇을 작성했습니다. CISA가 KEV 카탈로그에 등재했고, JFrog는 "최신 패치 버전도 여전히 취약"하다고 경고합니다.

💡 백엔드 개발자 포인트 — 핵심 교훈: (1) 사용자 입력을 exec()로 직접 실행하는 패턴은 절대 금지 — AI 파이프라인 도구라도 예외 없음, (2) "public" 엔드포인트에서 코드 실행을 허용하면 인증 여부와 무관하게 RCE, (3) Langflow를 사용 중이라면 즉시 1.9.0+으로 업그레이드하되 JFrog 권고도 확인. AI 개발 도구의 보안 감사를 CI/CD에 포함하세요.

💡 백엔드 개발자 포인트 — 자율실험실(Self-Driving Lab)은 로봇 제어 + AI 분석 + 데이터 파이프라인이 결합된 시스템입니다. 실험 데이터 수집 → AI 모델 추론 → 로봇 명령 생성의 이벤트 주도 파이프라인은 Kafka + CQRS 아키텍처와 유사합니다. 바이오/헬스케어 AI 백엔드 수요가 늘어나고 있으니, 도메인 확장을 고려해볼 시점입니다.
