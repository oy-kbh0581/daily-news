llm-d, CNCF 샌드박스 프로젝트로 기증 — K8s 네이티브 LLM 추론의 표준화 NEW

KubeCon Europe 2026에서 IBM Research, Red Hat, Google Cloud가 분산 LLM 추론 프레임워크 llm-d를 CNCF 샌드박스 프로젝트로 기증했습니다. NVIDIA, CoreWeave, AMD, Cisco, Hugging Face, Intel, Mistral AI가 공동 참여합니다. llm-d는 추론을 prefill/decode 단계로 분리하여 별도 Pod에서 실행하는 K8s 네이티브 아키텍처로, 모든 모델·가속기·클라우드에서 작동하는 벤더 중립 추론 청사진을 제공합니다.

💡 백엔드 개발자 포인트 — KServe(상위 제어 플레인)와 vLLM(하위 추론 엔진) 사이의 간극을 메우는 프레임워크입니다. EKS에서 LLM 서빙을 직접 운영하는 팀이라면, llm-d로 prefill/decode를 분리해 GPU 활용률을 높이고 레이턴시를 줄일 수 있습니다. K8s HPA 기반 오토스케일링과 자연스럽게 연동됩니다.

OpenClaw AI 에이전트 보안 위기 — 4일간 9 CVE, 마켓플레이스 12% 악성 CRITICAL

GitHub 스타 13.5만을 기록한 AI 에이전트 OpenClaw에서 4일간(3/18~21) 9건의 CVE가 공개되었습니다. 최고 CVSS 9.9점, 총 보안 권고 156건(128건 CVE 미할당). 샌드박스 우회로 자식 프로세스가 제한 없이 실행되는 취약점, "allow always" 기능 악용 RCE 등이 발견되었습니다. 공개 마켓플레이스 ClawHub에서는 2,857개 스킬 중 341개(12%)가 키로거·인포스틸러를 배포하는 악성 스킬로 확인되었습니다.

💡 백엔드 개발자 포인트 — AI 에이전트의 "자율 실행" 능력이 보안 위협의 새로운 차원을 열었습니다. (1) AI 에이전트에게 최소 권한만 부여(PoLP), (2) 셸 명령 실행은 화이트리스트 기반으로 제한, (3) 서드파티 스킬/플러그인은 코드 리뷰 후 사용, (4) 에이전트 실행 환경을 네트워크 격리된 컨테이너에서 운영하세요.
📊 기획자 포인트 — "앱스토어"와 같은 AI 에이전트 마켓플레이스에서 12%가 악성이라는 수치는 충격적입니다. AI 에이전트 기반 서비스를 기획할 때, 서드파티 플러그인 검증 프로세스(코드 서명, 보안 스캔, 리뷰)를 MVP 단계부터 설계해야 합니다.
