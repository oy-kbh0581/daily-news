LiteLLM 공급망 공격 — Trivy CI/CD 경유, 크리덴셜 탈취 3단계 악성코드 CRITICAL

3월 24일, 위협 행위자 TeamPCP가 Trivy GitHub Action을 먼저 장악한 뒤 LiteLLM CI/CD 파이프라인을 통해 PyPI 배포 토큰을 탈취했습니다. v1.82.7~v1.82.8에 삽입된 악성 코드는 3단계로 작동합니다: SSH 키·클라우드 크리덴셜·K8s 시크릿 수집 → K8s 권한 상승 및 횡적 이동 → systemd 백도어 설치. 약 3시간 만에 PyPI에서 격리되었으나, LiteLLM은 일일 340만 다운로드, 클라우드 환경 36%에 설치되어 있습니다.

💡 백엔드 개발자 포인트 — AI 게이트웨이가 공급망 공격의 주요 타깃이 되었습니다. LiteLLM 사용 중이라면 즉시 버전을 확인하세요. 대응 방안: (1) pip install 시 해시 검증(--require-hashes), (2) GitHub Actions에서 Third-party Action은 SHA 핀닝, (3) K8s에서 Pod Security Standards strict 모드 적용, (4) .env와 크리덴셜을 AWS Secrets Manager/KMS로 관리.
